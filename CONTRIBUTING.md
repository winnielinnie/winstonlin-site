# Contributing

Thanks for taking a look.

This repo is intentionally small and practical. If you want to contribute, the most helpful changes are:
- clearer docs
- bug fixes in the core logic
- tests for edge cases
- examples that make the project easier to reuse

## Ground rules

- Keep the scope tight.
- Prefer readability over cleverness.
- Add or update tests when changing behavior.
- If you expand the repo, explain why the extra complexity is worth it.

## Development notes

- Keep local examples easy to run.
- Avoid introducing infrastructure-heavy setup for small contributions.
- If a production-grade feature is intentionally left out, document that choice rather than hiding it.
